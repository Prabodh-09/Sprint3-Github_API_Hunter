**Sprint 3 - Prompts Log**

For this sprint, I used AI as a learning partner while building and debugging my project.

At the beginning, I wanted to understand the assignment clearly.

**Prompt:** "First explain this sprint in simple terms and tell me what exactly we are building."

This helped me understand that the project was about building a GitHub profile search application using Fetch API, Async/Await, DOM manipulation, and Battle Mode.

After that, I wanted to save time and build everything together.

**Prompt:** "We don't have time, let's do all the things at once. Give me steps one by one and also give code."

This helped me plan the project in steps and start quickly.

Then I asked for the project setup.

**Prompt:** "Give me the codes as well for step 1."

This helped me create the base structure of the project with HTML, CSS, and JavaScript files.

After setting up, I moved to API integration.

**Prompt:** "Step 2"

Here I learned how to fetch GitHub user data, use async/await, and render profile details like avatar, name, bio, join date, and portfolio.

Then I expanded the project.

**Prompt:** "Step 3"

This helped me learn endpoint chaining using repos_url, fetching repositories, formatting dates, and displaying the latest 5 repositories.

After that, I implemented the advanced feature.

**Prompt:** "Next step"

This was for building Battle Mode. I learned Promise.all() to fetch two users at the same time and reduce() to calculate total stars.

While testing, I faced issues.

One issue was with the name field.

**Prompt:** "The name is not visible after entering the data card section. It is showing no name."

From this, I understood that GitHub users may not have their real name filled, so I changed the logic to use username as fallback.

Another issue was data extraction.

**Prompt:** "Why can't we extract all the users, I tried for mine it is not coming."

This helped me understand API responses better and how usernames work.

Then I found that repositories were not showing properly.

**Prompt:** "The data is not extracting it well, it is not getting my repos."

This helped me debug with console.log() and understand public vs private repositories.

After that, I cleaned the UI.

**Prompt:** "Remove no portfolio, followers and following... I think they have not asked right?"

This helped me align the project exactly with sprint requirements.

Then I fixed Battle Mode UI.

**Prompt:** "Only when we click on battle mode then the username fields should appear, instead it is just staying."

This helped me implement conditional visibility using CSS and JavaScript.

While fixing it, I found another issue.

**Prompt:** "In CSS there is no .hidden one."

This helped me understand the importance of CSS utility classes in UI state management.

Later during GitHub push, I faced a git issue.

**Prompt:** "This is the error I have got"

This helped me fix the git merge conflict and successfully push my code.

Finally, before submission, I verified the mandatory requirements.

**Prompt:** "Have we done the loading state and 404 error state?"

This helped me confirm that both were implemented correctly and ready for the QA demo.

Overall, I used AI efficiently to understand the sprint, generate structure, debug problems, improve the project, and verify requirements instead of just copying code blindly.
